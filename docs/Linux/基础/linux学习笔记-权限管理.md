
Linux 是一个多用户、多任务的操作系统，其权限管理是保障系统安全和稳定运行的核心机制。它主要通过控制**用户**对**文件**和**目录**的访问来实现。

### 一、 用户与用户组基础

在深入权限之前，我们先理解 Linux 中的用户和用户组概念。

*   **用户 (User)**: 每个登录系统的用户都有一个唯一的用户名（Username）和用户ID（UID）。常见的有：
    *   `root`: UID 为 0 的超级用户，拥有系统的最高权限，可以执行任何操作。
    *   普通用户: 由管理员创建，用于日常操作，权限受限。
*   **用户组 (Group)**: 一组用户的集合，拥有唯一的组名（Groupname）和组ID（GID）。将用户分配到组的主要目的是为了方便批量授权。一个用户可以属于多个组。
*   **其他 (Others)**: 指既不是文件所有者，也不是文件所属组成员的所有其他用户。

这种“用户-用户组-其他”的三级结构是权限模型的基础。

---

### 二、 文件权限详解

Linux 中每个文件都与三个身份（所有者、所属组、其他人）和三种基本权限（读、写、执行）相关联。

#### 1. 基本权限类型

| 字母 | 数字表示法 | 含义 |
| :--- | :--- | :--- |
| `r` (read) | 4 | 读取文件内容的权限。 |
| `w` (write) | 2 | 修改、删除或重命名文件的权限。 |
| `x` (execute) | 1 | 将文件作为程序运行的权限。 |

#### 2. 查看文件权限

使用 `ls -l` 命令可以查看文件的详细信息，其中第一列就是权限字符串。

```bash
$ ls -l example.txt
-rw-r--r--  1 alice  developers  1024 Mar  6 10:00 example.txt
```

让我们分解这个输出：

*   **`-rw-r--r--`**: 这是权限字符串，共10位。
    *   第1位 (`-`): 表示文件类型。`-` 代表普通文件，`d` 代表目录，`l` 代表链接文件等。
    *   第2-4位 (`rw-`): 代表**文件所有者 (alice)** 的权限。这里表示所有者有读(`r`)、写(`w`)权限，但没有执行(`x`)权限。
    *   第5-7位 (`r--`): 代表**文件所属组 (developers)** 的权限。组成员只有读(`r`)权限。
    *   第8-10位 (`r--`): 代表**其他用户 (others)** 的权限。其他用户也仅有读(`r`)权限。
*   `1`: 硬链接数。
*   `alice`: 文件的所有者。
*   `developers`: 文件所属的用户组。

#### 3. 修改文件权限

*   **修改权限 (chmod)**:
    *   **字母方式**:
        *   `u` (user): 文件所有者
        *   `g` (group): 文件所属组
        *   `o` (others): 其他用户
        *   `a` (all): 所有人 (u, g, o)

        ```bash
        # 给所有者增加执行权限
        chmod u+x example.txt

        # 给所属组和其他人赋予读和写权限
        chmod go+rw example.txt

        # 移除其他人的所有权限
        chmod o= example.txt
        ```
    *   **数字方式 (八进制)**: 将三个权限组合起来计算。例如，要设置 `rwx` (读、写、执行)，就将对应的数字相加：4+2+1=7。要设置 `rw-` (读、写、不执行)，就是 4+2+0=6。

        ```bash
        # 设置所有者为 rwx(7)，组为 rw-(6)，其他人为 r--(4)
        chmod 764 example.txt
        ```

*   **修改所有者/所属组 (chown / chgrp)**:
    *   `chown new_owner file`: 更改文件所有者。
    *   `chgrp new_group file`: 更改文件所属组。
    *   `chown new_owner:new_group file`: 同时更改所有者和所属组。

### 三、 目录权限详解

目录的权限意义与文件略有不同，因为它本身是一个包含文件列表的特殊文件。

| 权限 | 对目录的意义 |
| :--- | :--- |
| `r` (读) | 可以列出目录下的内容（如使用 `ls` 命令）。 |
| `w` (写) | 可以在目录下创建、删除或重命名文件和子目录。 |
| `x` (执行) | 可以进入该目录（如使用 `cd` 命令）。如果缺少此权限，即使知道目录名也无法进入。 |

#### 实例分析

假设有一个目录 `/home/project`。

*   如果用户只有 `r` 权限：可以列出 `/home/project` 下有哪些文件和目录，但不能进入。
*   如果用户只有 `x` 权限：可以进入 `/home/project` 目录，也可以访问该目录下自己有权限的文件（前提是知道文件名），但无法列出目录内容。
*   如果用户有 `rx` 权限：这是最常用的目录访问权限，可以进入目录并列出内容。
*   如果用户有 `rwx` 权限：可以进入、列出内容，并且可以在该目录下创建、删除文件。

### 四、 用户权限管理

这部分主要涉及如何创建、管理和切换用户及用户组。

1.  **添加用户**: 使用 `useradd` 命令。
    ```bash
    sudo useradd -m -s /bin/bash newuser  # -m 创建家目录，-s 指定默认shell
    sudo passwd newuser # 为新用户设置密码
    ```

2.  **添加用户组**: 使用 `groupadd` 命令。
    ```bash
    sudo groupadd newgroup
    ```

3.  **将用户添加到组**: 使用 `usermod` 命令。
    ```bash
    # 将用户添加到附加组（不会影响其主组）
    sudo usermod -aG group_name username
    ```

4.  **切换用户**: 使用 `su` 或 `sudo` 命令。
    *   `su - username`: 切换到指定用户身份。
    *   `sudo command`: 以 root 或其他用户身份执行一条命令。

5.  **查看当前身份**: `whoami` 显示当前用户名，`groups` 显示当前用户所属的所有组。

### 总结

Linux 的权限体系是一个精巧的三层模型：

*   **身份**：所有者、所属组、其他人。
*   **对象**：针对文件（内容）和目录（容器）。
*   **权限**：读(r)、写(w)、执行(x)。

通过 `chmod`, `chown`, `chgrp` 等命令，系统管理员可以精确地控制谁能在系统中做什么，从而构建一个安全、有序的多用户环境。理解并熟练运用这些权限是进行 Linux 系统管理的关键技能。
